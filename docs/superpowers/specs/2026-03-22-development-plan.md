# 数据库桌面管理器开发计划 (MVP)

本文档基于《数据库桌面管理器设计文档》制定，旨在规划 MVP 阶段的开发里程碑、核心任务以及预计的交付节奏。

## 整体开发策略

- **技术栈**：Electron (主进程) + React (渲染进程) + TypeScript。
- **架构原则**：渲染层保持“薄”，所有数据库协议、安全存储、Agent 编排逻辑全部保留在本地核心服务（Main 进程），通过 IPC 暴露受控 API。
- **阶段划分**：分为基础设施、核心引擎、UI交互、智能 Agent 及测试发布 5 个主要里程碑。

---

## 里程碑 1：基础设施与脚手架搭建

**目标**：完成应用底座建设，打通进程间通信与本地数据存储。

- [ ] **项目初始化**
  - 使用 Electron + Vite + React 搭建基础工程。
  - 配置 TypeScript、ESLint、Prettier 及目录结构 (`src/main`, `src/renderer`, `src/core`, `src/shared`)。
- [ ] **IPC 通信桥梁建立**
  - 设计并实现 `preload.ts` 中的安全上下文 API (`contextBridge`)。
  - 建立 Main 进程与 Renderer 进程之间的类型安全事件总线。
- [ ] **本地存储服务模块**
  - 引入本地 SQLite（如 `better-sqlite3`）用于存储连接配置、历史记录。
  - 集成操作系统原生凭据管理器（使用 Electron 的 `safeStorage` 或 `keytar`），实现凭据的加密落盘与解密。

---

## 里程碑 2：数据库核心引擎与连接管理

**目标**：实现对多种数据库的底层访问能力与统一的生命周期管理。

- [ ] **统一数据库适配层**
  - 定义统一的数据库驱动接口（连接、断开、执行 SQL、获取元数据、中断查询）。
- [ ] **多数据库驱动实现**
  - 实现 MySQL 适配器。
  - 实现 PostgreSQL 适配器。
  - 实现 MSSQL 适配器。
  - 实现 ClickHouse 适配器。
- [ ] **连接管理器 (Connection Manager)**
  - 实现创建、测试、编辑、删除连接的业务逻辑。
  - 实现连接配置的持久化存储与加密凭据绑定。

---

## 里程碑 3：工作区 UI 与对象浏览器

**目标**：用户能够成功添加连接，并可视化地浏览数据库结构。

- [ ] **UI 框架与路由构建**
  - 引入组件库（如 TailwindCSS / Radix UI / AntD），搭建全局主界面布局。
  - 实现左侧连接列表管理面板。
- [ ] **对象浏览器 (Object Browser)**
  - 设计层级树组件（库 -> Schema -> 表 -> 视图 -> 字段）。
  - 对接底层获取元数据的 API，实现按需展开加载与手动刷新。
- [ ] **全局多 Tab 状态管理**
  - 设计多 Tab 工作区的数据结构，确保每个 SQL Editor/Chat Tab 强绑定到唯一且独立的连接 Session，避免串库。

---

## 里程碑 4：SQL 编辑器与查询执行

**目标**：用户可以编写 SQL，安全地执行并查看结果。

- [ ] **SQL 编辑器组件**
  - 集成 Monaco Editor，配置 SQL 语法高亮和基本的代码补全。
- [ ] **查询执行器 (Query Executor) 完善**
  - **大结果集防御**：在执行前，对于未显式包含 `LIMIT` 的查询，通过 AST 或正则自动注入 `LIMIT 1000`（或游标截断），避免内存溢出。
  - **长耗时查询管理**：提供获取当前执行状态的机制，实现中断/取消查询（Kill Query）功能。
- [x] **结果集与历史展示**
  - 实现支持虚拟滚动的结果数据表格（Data Grid），以承载大数据量的渲染。
  - 实现查询历史面板，记录执行耗时、状态及 SQL，支持快捷回放。

---

## 里程碑 5：Chat Agent 与大模型集成

**目标**：接入大模型，实现自然语言到 SQL 的受控生成。

- [x] **LLM 基础设施与设置**
  - 开发系统设置页：提供 OpenAI API 和 Anthropic API 的密钥配置入口。
  - 增加隐私边界提醒，在用户首次配置或使用时弹窗，告知发送至云端的数据范围（仅表结构，不含业务数据）。
- [ ] **Schema 轻量级索引机制**
  - 实现后台任务：在连接建立后，轻量化提取当前库的表名、字段名和注释，并在本地构建搜索索引。
  - 实现按需注入逻辑：根据用户 Prompt 提取关键词，检索最相关的 1-5 张候选表，动态拼接 DDL 进入上下文，支持复杂的多表关联生成。
- [ ] **Agent 对话 UI 面板**
  - 实现 Chat 界面，支持用户输入问题，展示 Agent 的“思考过程”、生成的 SQL 及风险说明。
- [ ] **安全控制器 (Security Controller)**
  - 简单的 SQL 风险分类：基于正则/关键词区分只读（如 `SELECT`）与危险操作（如 `UPDATE`, `DROP` 等）。
  - 实现自动执行逻辑：只读操作自动流转至查询执行器。
  - 实现安全卡点：危险操作强制弹出“执行确认弹窗”。

---

## 里程碑 6：测试、体验优化与发布

**目标**：打磨边界场景，交付可用的桌面安装包。

- [ ] **错误处理与异常体验优化**
  - 处理并美化连接失败（如网络不通、认证失败）、SQL 语法错误等提示。
  - 优化 Agent 生成失败或上下文超出时的降级引导（切换至手写模式）。
- [ ] **应用安全与性能加固**
  - 测试 `safeStorage` 在 Windows 和 macOS 下的加解密稳定性。
  - 检查内存泄漏，尤其是多 Tab 和大数据量表格的销毁逻辑。
- [ ] **打包与发布**
  - 配置 `electron-builder`。
  - 调整 App 图标、打包元数据。
  - 输出针对 macOS (dmg/arm64) 和 Windows (exe) 的安装包。
